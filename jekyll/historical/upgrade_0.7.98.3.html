Upgrading to new version

0.7.98.2 -> 0.7.98.3

Usage of /HILIGHT -color, /SET hilight_color and /SET hilight_act_color has changed. See NEWS file for more information.

Statusbar is now configurable from theme file, all /SET statusbar_xxx settings have been removed.

0.7.97 -> 0.7.98

Settings in config file have changed a bit. Irssi can convert your old configuration file to new format properly, but first time you run new Irssi, you shouldn't do things like /RELOAD .. Would be better if you just did /SAVE and restart irssi.

Theme files have changed some. If you haven't made modifications to the default theme, you could just rm ~/.irssi/default.theme. I've also written a small guide for upating .97 themes to .98. This causes the <@ nick> problem (space between @ and nick).

Old scripts are incompatible. There's too many changes to list, but biggest change is that instead of $object->values()->{xxx} you can now use $object->{xxx} directly.

IPv6 changes don't seem to work properly with some operating systems.. I'll have to investigate more, but for now you can just explicitly specify when you want to connect to IPv6 server with /CONNECT -6 or updating server configuration with /SERVER ADD -6 irc.server. Or if you use only IPv6 servers you could do /SET resolve_prefer_ipv6 ON
