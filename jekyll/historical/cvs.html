CVS

Needed software

To build irssi from CVS, you need to have autoconf, automake and libtool installed. These come with most distributions. You also need either GLib 1.2 or GLib 2.0.

Getting it

First login to cvs, you only need to do this the first time you check out the sources:

cvs -d:pserver:anonymous@cvs.irssi.org:/home/cvs login

Just hit enter as the password.

And then check out the sources:

cvs -z3 -d:pserver:anonymous@cvs.irssi.org:/home/cvs co -P irssi

NOTE: Because of some bug in CVS 1.11.2 server, if you connect with older clients and use -z option, it hangs at the end. Everything is transferred, but it just gets stuck there. So your options are: a) ignore and just hit Ctrl-C, b) upgrade to CVS 1.11.2, c) don't use -z option, d) try to get CVS authors to fix it - this is not a new problem.

To update sources later, use this command in irssi directory:

cvs -z3 update -dP

and to compile it, run ./autogen.sh

xirssi

Just like above with irssi, except checkout xirssi instead:

cvs -d:pserver:anonymous@cvs.irssi.org:/home/cvs login
cvs -z3 -d:pserver:anonymous@cvs.irssi.org:/home/cvs co -P xirssi

You need to have irssi compiled and configured with --with-glib2 option. xirssi also needs to find irssi's sources, by default it looks into ../ and ../irssi/ directories, but if it's not there, you need to give --with-irssi=<path> option to configure.

Troubleshooting

Make sure you have all the required software and new enough versions of them. These are known to work:

  • autoconf 2.13
  • automake 1.4
  • libtool 1.3.5
  • GNU make 3.77 - non-GNU makes won't work
  • GLib 1.2.6

If you get some errors about AC_CHECK_CURSES or AM_CONFIG_HEADER or something, you're missing some of the .m4 files. They should exist in /usr/share/aclocal/ directory, if you have them somewhere else, specify them with export ACLOCAL_FLAGS='-I /dir -I /dir2' before running autogen.sh. These files are required:

  • libtool.m4 (in libtool package)
  • header.m4 (in automake package - there's lots of others too, but at least this one)
